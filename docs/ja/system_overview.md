# システム概要

## プロジェクトの目的

このプロジェクトは、Ubuntu 24.04 上で C 言語によるシンプルな TCP/IP 通信制御システムを作成する学習用プロジェクトです。

いきなり実装に入るのではなく、設計書を先に整備し、その後 TCP サーバ、Windows GUI クライアント、ログ保存、テスト、GitHub Actions へ段階的に拡張していきます。

## 背景と学習目標

このリポジトリは、コードだけでなく設計の流れも見せられる GitHub ポートフォリオとして育てることを想定しています。

学習目標は以下です。

- TCP サーバの基本的な動作流れを理解する。
- 実装前に簡単なテキストベースの通信仕様を定義する。
- 実装フェーズを小さく分けて確認しやすくする。
- 今後 Codex が読み違えにくい設計文書を残す。
- 産業機器や組み込み通信を意識した小さな通信例へ発展させる。

## 想定ユースケース

このシステムは、小規模な制御・監視システムを想定しています。

Windows PC クライアントが TCP/IP 経由で Ubuntu TCP サーバへコマンド文字列を送信します。サーバは内部状態を持ち、コマンドに応じて状態を更新し、応答を返します。将来的にはログ保存やセンサ値シミュレーション、CAN 連携へ拡張します。

## システム構成

[Windows Client]
  - コマンド送信

  - 将来的に GUI 表示

## コンポーネントの役割

### Windows クライアント

- Ubuntu TCP サーバへ接続する。
- 1 行に 1 コマンドを送信する。
- サーバからの応答を表示する。
- 将来的には Windows GUI クライアントとして実装する。

### Ubuntu TCP サーバ

- TCP 待ち受けソケットを開く。
- クライアント接続を受け付ける。
- 行単位のコマンド文字列を受信する。
- 対応コマンドを解析する。
- 必要に応じて内部状態を更新する。
- コマンドごとに 1 つの応答を返す。
- 将来的にログ保存と設定ファイルに対応する。

### 将来の CAN / センサシミュレータ

- センサ値や CAN 風のデータを提供する。
- 組み込み通信を意識したサンプルへ発展させる。
- Phase 2 および Phase 3 の実装対象外とする。
